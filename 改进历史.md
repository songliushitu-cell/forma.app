## 2025-04-15  初始批量改进（人工审阅后实施）

**来源文档：** 1--模块七正课.doc / 020.模块八直播.docx / 前压缩和后压缩训练方案_整理版.pdf 等

- **[A] 评估逻辑**
  - `buildRotationIsolationResult`：左右差阈值从 5° 改为 10°（`nearThreshold` → `threshold`）
  - `buildRouteIsolationAssessment`：新增 `hasMeaningfulLateral`（任意测试差值 ≥ 10°）和 `trainingStage`（三阶段入口判定：Stage1 = SLR<45且肩屈<90；Stage3 = 有明显侧倾；Stage2 = 其余）

- **[B] 训练阶段提示**
  - `STAGE_TIP[1]`：新增退出目标（肩屈>90°、SLR>45°、髋屈>90°）
  - `STAGE_TIP[2]`：新增「允许左右不对称依然存在」说明 + 退出目标（肩屈120°、SLR 60-70°、髋屈120°）
  - `STAGE_TIP[3]`：按 `subType` 动态展示三套内容（基线型/横向型/斜轴型），原「固定一句话」改为「要减少/要建立」结构化训练方向；Stage3 提示框从隐藏改为常显

- **[C] 报告内容**
  - `posterior_baseline.avoid`：新增「不要拉伸绳肌/臀部（后压缩人群后侧已过度拉长）」、「不要大量臀桥（诱发腰椎代偿）」
  - `anterior_baseline.avoid`：新增「不要孤立激活臀部而不先解决前侧拉力（前侧不松开臀肌发不上力）」
  - `posterior_lateral.direction`：改为「要减少/要建立」结构，指向右腿单侧承重动作
  - `posterior_oblique.direction`：明确「两个维度都要处理，不能只压高度」
  - `anterior_lateral.direction`：改为「要减少/要建立」结构，唤醒后侧臀肌
  - `anterior_oblique.direction`：斜向双维度处理，强调前侧松开才能臀肌发力

- **[D] 备用（暂不实施）**
  - 「脏测试（Dirty Measures）」警告：相似测试结果可能有完全相反的根本原因，需综合评估。已知但暂未在报告 UI 中体现。
  - ISA（胸骨下角）宽窄对 SLR 结果的影响：不影响分型，但可作为报告注释。

## 2026-04-24  报告共情化与分区强化改版（v1.1）

**来源文档：** 用户确认的《报告共情化与分区强化改版方案（v1.1 定稿）》

- **[A] 报告结构**
  - `renderIsolatedFinalReport`：最终报告顺序调整为「结果总览 → 偏移方向 → 你平时怎么发现 → 过渡短句 → 身体感觉 → 外形变化 → 你的身体优势 → 调整方向 → 3 天微行动清单 → 测试数据 → 报告帮助反馈 → 免责声明」
  - 结果总览新增 `final-report-badges` 专属识别标签，按 `subType + compKey` 生成双徽章

- **[B] 共情内容**
  - 新增 `DAILY_LIFE_COPY_VERSION = 1.1.0`
  - 新增 `DAILY_LIFE_COPY_BY_SUBTYPE` / `DAILY_LIFE_COPY_BY_COMPRESSION`，以 JS 常量形式维护版本化生活场景文案
  - 新增 `buildDailyLifeCards`：按 `oblique > lateral > baseline` 优先级拼装左右维度，再叠加前后维度；问卷仅参与排序，不改写分型
  - 新增 `REPORT_ADVANTAGE_COPY` / `MICRO_ACTION_COPY`，对应“你的身体优势”和“3 天微行动清单”

- **[C] 交互与存储**
  - 新增本地存储键：
    - `pelvic_daily_life_feedback_v1`
    - `pelvic_report_helpfulness_v1`
    - `pelvic_micro_actions_v1`
  - 新增 `buildReportFeedbackState` 及相关读写函数，保存生活卡片“对得上”反馈、报告帮助反馈、微行动勾选状态
  - 报告区新增事件委托，确保保存后的历史报告恢复后仍可继续勾选与反馈
  - `buildCurrentReportSnapshot` / `applyReportSnapshot` 扩展保存与恢复：
    - `copyVersion`
    - `reportBadgesHtml`
    - `reportId`
    - `dailyLifeFeedbackState`
    - `reportHelpfulnessState`
    - `microActionState`

- **[D] 样式与安全**
  - 新增样式类：
    - `report-panel-empathy`
    - `daily-life-card`
    - `daily-life-icon`
    - `report-transition-note`
    - `report-advantage-card`
    - `micro-action-item`
    - `report-feedback-chip`
  - 红旗项触发时，隐藏“你平时怎么发现 / 你的身体优势 / 你现在怎么调整 / 3 天微行动清单 / 报告帮助反馈”，在原日常板块位置改为固定安全提醒
  - 报告底部新增标准免责声明
